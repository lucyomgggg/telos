# 設計原則
1. シンプルであること
複雑さはバグと遅延の温床。判断に迷ったらシンプルな方を選ぶ。
2. スケールを前提とすること
最初から大きく作らない。ただし、大きくなれない設計もしない。
3. 許容度の高い設計
想定外のデータ・要件・変化を吸収できること。アーキテクチャで未来をガチガチに決定しない。
4. 単一責任・単一窓口
各レイヤーの責務を明確に分け、接続先を一本化する。複数の経路は混乱の原因。
